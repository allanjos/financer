Application configuration

Application configuration Configuration resides in the config directory located on the current directory where application is executed. It can be generated by application accessing Preferences dialog and saving your configurations. The format is: 
<?xml version="1.0" encoding="UTF-8"?>
<config>
  <database>
    <connection host="localhost" port="5432" dbname="financer" user="postgres"
                password="postgres" sslmode="disable" filepath="financer.db" type="SQLite"/>
  </database>
  <language id="pt-BR"/>
  <rfid port="COM4" speed="57200"/>
  <logger>
    <log.entry name="system" level="0" enabled="true"/>
    <log.entry name="database" level="0" enabled="true"/>
  </logger>
</config>


Not all properties are needed on all the cases.

Database configuration

The database used is PostgreSQL. The support for MySQL is being added. SQLite support is being planned too. 
    cd <FINANCER_DIR>/database
Create the database. Configure the script variables for your needs before executing it. 
    sh financer-db.sh
